# macOS 联调说明

## 1. 当前可测能力

在 macOS 上，本测试版当前可验证以下内容：

- LemonTea Crow HTTP API 正常工作
- HoneyTea 与 LemonTea 可通过 TCP 模式建立长连接并自动重连
- HoneyTea 与 LemonTea 可通过 WebRTC DataChannel 模式建立控制链路
- 服务端下发 shell 命令，客户端执行并返回结果
- 服务端浏览客户端文件目录
- 服务端写入客户端文件
- 客户端插件按 manifest 自动拉起
- 客户端摄像头插件枚举设备
- 如果系统安装了 imagesnap，则支持单帧抓拍
- LemonTea-vue3 可正常构建并提供图形控制台

## 2. 当前受限能力

### 2.1 WebRTC 视频流

本轮已经落地 WebRTC DataChannel 控制链路，但还没有把摄像头升级为连续视频轨传输。

当前 WebRTC 已覆盖：

- 控制消息
- 命令转发
- 文件接口请求
- 插件调用请求

当前未覆盖：

- 连续视频流
- WebRTC 媒体轨推送

### 2.2 角度传感器

通用 macOS 设备没有统一公开的角度传感器 API。当前 `angle_sensor_plugin.py` 已改为自动探测模式，会尝试扫描系统公开信息中的角度/方向相关字段；未探测到时才返回 `supported=false`。

### 2.3 光线传感器

通用 macOS 设备也没有稳定统一的环境光传感器接口。当前 `light_sensor_plugin.py` 已改为自动探测模式，会尝试通过 `ioreg` 探测环境光相关控制器和数值；未探测到时才返回 `supported=false`。

### 2.4 连续视频流

当前摄像头插件支持：

- 设备枚举
- 条件性单帧抓拍

当前还不支持：

- 连续视频流
- 通过 WebRTC 发送视频轨

## 3. 如果你要换测试子进程

如果你希望在 macOS 上做更稳定的硬件能力联调，建议把这两个传感器子进程替换为以下更容易落地的子进程：

1. 电池信息读取
2. CPU / 内存 / 磁盘监控
3. 网络状态采集
4. 音频输入设备枚举
5. USB 设备枚举

这些能力在 macOS 上更容易通过系统命令或公开 API 暴露成插件。
